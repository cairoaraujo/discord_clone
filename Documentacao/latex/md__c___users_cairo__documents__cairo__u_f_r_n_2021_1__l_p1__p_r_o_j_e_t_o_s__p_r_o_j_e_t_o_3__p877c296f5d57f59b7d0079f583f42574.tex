O Concordo possui as seguintes funções\+:\hypertarget{md__c___users_cairo__documents__cairo__u_f_r_n_2021_1__l_p1__p_r_o_j_e_t_o_s__p_r_o_j_e_t_o_3__p877c296f5d57f59b7d0079f583f42574_autotoc_md4}{}\doxysubsection{Funções de Usuário\+:}\label{md__c___users_cairo__documents__cairo__u_f_r_n_2021_1__l_p1__p_r_o_j_e_t_o_s__p_r_o_j_e_t_o_3__p877c296f5d57f59b7d0079f583f42574_autotoc_md4}

\begin{DoxyItemize}
\item {\bfseries{create-\/user\+:}} a função espera os parâmetros \char`\"{}\+Email\char`\"{} \char`\"{}\+Senha\char`\"{} e \char`\"{}\+Nome\char`\"{}, respectivamente. O email e senha não devem possuir espaços, porém o Nome pode conter. Caso o usuário não informe algum dos parâmetros, o mesmo ficará vazio, porém o cadastro ainda funcionará.
\item {\bfseries{login\+:}} Espera-\/se que o usuário informe o \char`\"{}\+Email\char`\"{} e \char`\"{}\+Senha\char`\"{}, respectivamente. Caso o usuário não exista, o login retornará erro. Caso contrário, o usuário entrará no Concordo. O sistema aceita somente 1 usuário por vez, então caso haja o comando {\bfseries{login}} já com algum usuário logado, a função retornará erro.
\item {\bfseries{disconnect\+:}} Irá desconectar o usuário logado. Caso não haja usuário logado, a função retornará erro.
\item {\bfseries{quit\+:}} Sairá imediatamente do Concordo.
\end{DoxyItemize}\hypertarget{md__c___users_cairo__documents__cairo__u_f_r_n_2021_1__l_p1__p_r_o_j_e_t_o_s__p_r_o_j_e_t_o_3__p877c296f5d57f59b7d0079f583f42574_autotoc_md5}{}\doxysubsection{Funções de Servidor\+:}\label{md__c___users_cairo__documents__cairo__u_f_r_n_2021_1__l_p1__p_r_o_j_e_t_o_s__p_r_o_j_e_t_o_3__p877c296f5d57f59b7d0079f583f42574_autotoc_md5}

\begin{DoxyItemize}
\item {\bfseries{Create-\/server\+:}} Função que cria um novo servidor, cujo recebe como parâmetro apenas o nome do servidor fornecido pelo usuário. É necessário que haja algum usuário logado, caso contrário, retornará um erro.
\item {\bfseries{Enter-\/server\+:}} Função que fará o usuário logado entrar em determinado servidor já criado. Para usar a função, o usuário deve informar o {\bfseries{nome do servidor}} e o {\bfseries{código convite}}, respectivamente. Caso o servidor específico não possua código convite, não é necessário informar esse parâmetro.
\item {\bfseries{Leave-\/server\+:}} a função leave-\/server irá remover o usuário logado do servidor atual. Assim, caso o usuário deseje acessar o servidor novamente, será preciso usar a função enter-\/server.
\item {\bfseries{Remove-\/server\+:}} Remove o servidor informado. Recebe como parâmetro o {\bfseries{nome do servidor}} que deseja-\/se remover. Retornará mensagem de erro caso o servidor informado não exista.
\item {\bfseries{List-\/participants\+:}} Essa função imprimirá a lista de participantes do servidor atual onde o usuário está conectado.
\item {\bfseries{List-\/servers\+:}}\+: Imprime todos os servidores já criados, com suas respectivas descrições. Caso a descrição esteja vazia, não imprimirá a descrição.
\item {\bfseries{Set-\/server-\/desc\+:}} Com o usuário logado, essa função poderá adicionar uma descrição para o servidor informado pelo usuário. O parâmetro da funação receberá do usuário respectivamente o {\bfseries{nome do servidor}} e a {\bfseries{descrição}}. A descrição pode ser alterada por qualquer usuário e, por default, a descrição de um servidor é vazia.
\item {\bfseries{Set-\/server-\/invite-\/code\+:}} Com usuário dono do servidor logado, essa função criará um código-\/convite de determinado servidor. Assim, a única forma de acessar o servidor é informando esse código. O usuário deve informar, respectivamente, o {\bfseries{nome do servidor}} e o $\ast$$\ast$código convite desejado.
